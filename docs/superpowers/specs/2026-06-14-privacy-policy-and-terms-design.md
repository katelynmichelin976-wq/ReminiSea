## 1. 背景与范围

上架 App Store / Google Play 要求 app 必须提供：
- 隐私政策（Privacy Policy）
- 用户协议 / 服务条款（Terms of Service）
- 首登前明确同意流程

### 服务定位（决定合规框架）

- **上架地**：仅 US App Store（不上架中国区 / Google Play 中国版）
- **目标用户**：全球华人社区，以**海外华人**为主
- **营销范围**：不在中国大陆主动推广、不接受中国大陆境内付费转账；国内自发使用不排斥但**非目标受众**
- **支持语言**：多语言（zh-CN/zh-Hant/en/es/ja），符合"面向全球而非特定地区"特征
- **运营主体**：境外个人开发者，无中国大陆运营实体

### 合规框架（基于上述定位）

| 法律/规则 | 是否适用 | 处理 |
|---|---|---|
| **Apple App Review Guideline 5.1** | 必须 | ✓ 提供隐私政策 URL + 同意机制 + 删除流程 |
| **App Store Privacy Nutrition Labels** | 必须 | App Store Connect 后台填写，跟本设计独立 |
| **COPPA**（美国，<13 岁儿童）| 必须 | 一句不针对儿童声明 |
| **CCPA**（加州州法）| 不触发阈值（个人小开发者）| 礼貌性披露用户权利 |
| **GDPR**（欧盟）| 不触发（未主动针对欧盟）| 备注国际用户提示，不做强承诺 |
| **PIPL §38 跨境合同/认证**（中国）| **不触发**（未主动针对中国大陆）| 服务器位置作事实披露，不走 PIPL 同意框架 |
| **ICP 备案**（中国大陆）| **不需要**（无境内运营）| 不做 |

### 2.1 目标

- 上架基础合规：提供可访问的中文 PP/ToS
- 用户明示同意：注册/登录前 checkbox 强制勾选
- 文档可独立修订（外部 HTML，不依赖 release）
- 同意状态记录到本地 + 云端 sync_config

### 2.2 非目标

- ❌ 英文版（P2）
- ❌ 律师定稿（P2）
- ❌ PIPL 标准合同 / 跨境合规细节（P2）
- ❌ 老用户回溯同意 migration（P2）
- ❌ 付费/退款条款（P3）
- ❌ 儿童保护专章（app 不针对儿童，简短声明即可）

## 4. UI 实现

### 4.1 文件结构

| 文件 | 改动 |
|---|---|
| `privacy.html` | 新建（GitHub Pages 根目录），约 200 行 HTML + 内联 CSS |
| `terms.html` | 新建，约 200 行 |
| `index.html` | 登录/注册 form 加 checkbox + label + 链接 + submit disabled 逻辑（~40 行）+ i18n 新 key |
| `tests/_pw_consent_checkbox.js` | 新建（~80 行，~10 断言）|

### 4.2 Checkbox UI 设计

注册/登录表单底部（`<form>` 内 submit 按钮之上）插入：

```html
<label class="consent-label">
  <input type="checkbox" id="consent-checkbox" required>
  <span data-i18n-html="consent_label">
    我已阅读并同意
    <a href="https://katelynmichelin976-wq.github.io/ReminiSea/privacy.html" target="_blank" rel="noopener">《隐私政策》</a>
    和
    <a href="https://katelynmichelin976-wq.github.io/ReminiSea/terms.html" target="_blank" rel="noopener">《用户协议》</a>
  </span>
</label>
```

JS 逻辑：
- 初始 submit 按钮 `disabled`
- checkbox change → 重算 disabled 状态
- 注册和登录共用一个 form / 不同 form 各加一份

### 4.3 i18n key

新增 5 个 key × 5 语种（zh-CN/zh-Hant/en/es/ja，但 PP/ToS 链接全部指向中文版 — P1 范围）：

| key | zh-CN |
|---|---|
| `consent_label` | "我已阅读并同意" |
| `consent_privacy` | "《隐私政策》" |
| `consent_and` | "和" |
| `consent_terms` | "《用户协议》" |
| `consent_required_hint` | "请先勾选同意才能继续" |

### 4.4 同意状态持久化

- 本地：`yh:v1:user:consentAt`（ISO 时间戳）+ `yh:v1:user:consentVersion`（"v1"）
- 云端：在 sync_config 加 `consent_at` + `consent_version` 字段（pull/push 时自动同步，跨设备一致）
- 老用户：本次首登时（P1 上线后）再次勾选记录；现状已注册用户**视为已同意旧版本**（不强制踢出 — 平衡）
