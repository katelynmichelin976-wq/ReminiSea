# 牌组管理页设计文档

**日期：** 2026-06-05  
**状态：** 定稿

---

## 概述

新增独立「牌组」页面，作为底部 Tab Bar 的第二个入口。将现有首页的牌组管理功能（新建、导入、导出、删除、重命名）集中到此页，同时整合云端个人牌组下载/同步及未来精选牌组功能。

---

## 导航结构变更

### Tab Bar：3 → 5 个入口

| 位置 | 图标 | 标签 | 说明 |
|------|------|------|------|
| 1 | 房屋 | 首页 | 现有，不变 |
| 2 | 卡片堆叠 | 牌组 | **新增**，进入牌组管理页 |

| 4 | 柱状图 | 统计 | **新增入口**（原为子页面） |
| 5 | 人像 | 我的 | 现有，不变 |

Tab 激活状态：顶部 2.5px 红色横杠 + 图标/文字变红，横杠与图标之间有 10px 间距。

### 牌组页导航

- **入口**：Tab Bar「牌组」
- **类型**：推入式子页面（非模态），顶部导航栏含「‹ 首页」返回按钮
- **无底部 Tab Bar**（子页面规范）

## 牌组页详细设计

### 顶部导航栏

```
‹ 首页        牌组        ＋
```

- 左：红色返回按钮「‹ 首页」
- 中：标题「牌组」（居中，粗体）
- 右：红色「＋」按钮，点击弹出底部 action sheet

### ＋ 按钮 Action Sheet（复用现有实现）

```
┌──────────────────┐
│ ＋ 创建空牌组     │
│ ↑ 导入文件        │
├──────────────────┤
│      取消         │
└──────────────────┘
```

复用 `openActionSheet()` / `createEmptyDeck()` / `importYhspack()` 现有逻辑，不改动。

### 分段控制器

三段：**本地 · 云端 · 精选**

---

## 本地 Tab

### 牌组列表

- 每行：牌组名 + 右箭头（点击进入浏览模式）
- 左滑展开操作按钮（复用现有 swipe 逻辑）：
  - 重命名（蓝色）→ `renameDeck()`
  - 导出（深蓝）→ `exportDeck()`
  - 删除（红色）→ `deleteDeck()`
- 底部提示文字：「左滑牌组可重命名、导出或删除」

### 从首页迁移

- 移除首页「我的牌组」区段头部的「＋ 新建」按钮（`advanced-only`）
- 首页牌组列表保留，功能不变（选中 + 练习）
- 新建/导入/导出/删除/重命名 操作统一收归牌组页

---

## 云端 Tab（我的个人牌组）

需登录后可用。牌组按状态分三区显示：

### 下载中

```
蔬菜水果                    暂停
152 / 400 张
████████░░░░░░░░░░░░░░░   (进度条)
```

- 进度条（蓝色，4px 高，`border-radius: 2px`）
- 「X / Y 张」实时更新
- 「暂停」按钮 → `toggleDownloadPause()`，暂停后变「继续」

### 已下载

| 状态 | 副文字 | 按钮 |
|------|--------|------|
| 已同步 | `N 张 · 已同步` | 同步（灰色 pill） |
| 有更新 | `云端有 N 张新卡` | 更新（琥珀色 pill） |

- 「同步」→ `doCloudDeckAction()` 触发上传检查
- 「更新」→ 拉取云端新增卡片

### 仅云端（未下载）

- 副文字：`N 张 · 未下载`
- 「下载」按钮（蓝色 pill）→ `doCloudDeckAction()`，立即替换为进度条

## 精选 Tab

预留设计，当前显示占位内容。分两组：

- **官方精选**：系统预制牌组，标签「官方」（绿色），下载/已下载（✓）
- **热门推荐**：社区共享牌组，标签「热门」（橙色），下载按钮

> 精选 Tab 数据源与后端接口待设计（publishDeck 功能边界确定后实现）。
