# iOS 专属 UI 重设计规格

日期：2026-06-18

范围：设计规格，不改代码。用于后续 iOS PWA / Capacitor / WKWebView 版本的 UI 评审和实现。

## 设计判断

产品定位是家庭成员协助记忆训练的移动应用，不是通用学习打卡工具。iOS 版本应优先让照护者、家属和记忆训练对象快速进入“今天练什么、怎么继续、声音/图片是否可用”的核心路径。

采用“温和医疗 + 家庭相册”的方向：克制、清晰、可触控，但保留记忆水波和老照片的情绪线索。避免模板化 SaaS 卡片堆叠，也避免过度儿童化或游戏化。

## 设计系统草案

### 色彩

基于 `uiuxpro` 的 healthcare / family 推荐，主色采用平静蓝绿，辅以记忆琥珀：

- `--ios-primary: #0891B2`：主要行动、当前进度、焦点环。
- `--ios-primary-pressed: #0E7490`：按钮按下态。
- `--ios-accent: #D97706`：记忆提示、今日重点、家庭相册标记。
- `--ios-success: #059669`：同步完成、保存成功。
- `--ios-bg: #ECFEFF`：页面底色，减少纯白眩光。
- `--ios-surface: #FFFFFF`：卡片和 sheet。
- `--ios-text: #164E63`：正文主文本。
- `--ios-muted: #64748B`：辅助说明。
- `--ios-border: #A5F3FC`：分隔和轻边框。
- `--ios-danger: #DC2626`：删除、退出、不可恢复动作。

深色模式需单独设计，不应简单反相。文字对比度：正文至少 4.5:1，辅助文本至少 3:1。

### 字体

优先级：

1. iOS 系统字体 `-apple-system`，确保 Dynamic Type、中文渲染和性能稳定。
2. 若后续接受内置字体，可考虑 `Lexend` 用于英文辅助标签、`Nunito Sans` 用于柔和正文；中文仍走系统字体。
3. 不建议继续依赖线上 Google Fonts 作为 iOS 包首屏关键资源。

字号基线：

- 大标题：30-34pt，行高 1.15。
- 屏幕标题：24-28pt，行高 1.2。
- 卡片标题：18-20pt，Semibold。
- 正文：16-18pt，行高 1.5。
- 辅助说明：14-15pt，不低于 14pt。
- 数据/标签：12-13pt，仅用于非关键短标签。

### 间距和触控

- 全局触控目标：最小 `44x44pt`。
- 相邻触控目标间距：至少 `8pt`。
- 页面水平边距：小屏 `18pt`，标准机 `20pt`，大屏 `24pt`。
- 卡片内边距：`16/20/24pt` 三档，不使用随机值。
- 固定底部操作区必须叠加 `safe-area-inset-bottom`。
- 顶部内容必须避开状态栏、Dynamic Island 和导航栏。

## 信息架构

### 底部导航

iOS 版本只保留 4 个顶级入口：

- 今日：默认首页，展示下一组训练、最近进度、继续练习。
- 牌组：管理家庭牌组、示例牌组、导入/导出。
- 记录：学习进度、SRS 状态、同步状态。
- 我的：账号、隐私、设置、帮助、诊断支持模式。

诊断入口不得放在版本号隐藏连击中作为生产默认路径。若保留，应移到“我的 > 帮助 > 支持码/诊断”，并要求用户主动确认。

### 首次打开

空状态必须提供可完成路径：

1. 选择语言。
2. 说明隐私和本地/云同步关系。
3. 选择开始方式：使用示例牌组、导入家庭牌组、登录同步。
4. 完成第一轮 3-5 张卡片训练。

onboarding 可跳过，但跳过后首页仍要有清晰的下一步。

## 核心屏幕规格

### 今日

目标：用户 5 秒内知道“现在按哪里开始”。

- 顶部使用一句明确状态文案，例如“今天先练 8 张熟悉的照片”。
- 主 CTA 固定为“开始今天训练”，只有一个主按钮。
- 进度以温和环形或水波进度表达，不用复杂排行榜。
- 同步状态放在次级区域，不抢主行动。
- 空状态提供“使用示例牌组”和“导入家庭牌组”两个清晰按钮。

### 训练页

目标：降低误触，让记忆对象专注卡片。

- 卡片图片/文字占据首屏视觉中心。
- “认识 / 模糊 / 不认识”按钮需横向或纵向明确分区，均满足 44pt。
- 语音播放、图片放大、提示语为辅助控件，不与评分按钮竞争。
- 答案反馈使用 150-250ms 的轻量过渡，不使用装饰性大动画。
- 横屏仍可显示提示 overlay，但正式 iOS 设计应尽量保证竖屏可用。

### 牌组

目标：让家属能安全管理内容。

- 列表项显示封面、名称、卡片数、最近同步状态。
- 删除、覆盖导入、清空数据必须二次确认，并提供明确后果。
- 导入流程需要进度和错误恢复，不只显示技术错误。
- 个人媒体隐私状态应可见，例如“仅自己可见 / 已同步到家庭云端”。

### 记录

目标：解释训练状态，不制造焦虑。

- 使用“今天完成”“连续练习”“需要复习”三类高层信息。
- SRS 细节默认折叠，给照护者查看，不作为第一层压力。
- 图表不依赖颜色单独表达含义，必须有标签或文本摘要。

### 我的

目标：把账号、隐私、帮助和支持入口集中。

- 登录状态、云同步状态、数据导出/删除路径必须可见。
- 隐私政策、用户协议、数据删除说明不能藏在二级深处。
- 诊断支持模式需要明确说明会收集哪些日志、如何关闭。

## 交互和无障碍要求

- 支持 VoiceOver：所有图标按钮必须有可读标签。
- 支持 Dynamic Type：大字号下按钮不截断，卡片可以增高。
- 支持 reduced motion：训练反馈和页面转场降级为淡入淡出。
- 不依赖 hover；所有关键动作必须可点击/触控完成。
- 表单使用正确输入类型：邮箱、数字、搜索等触发对应键盘。
- Toast 使用 `aria-live="polite"` 或等价机制，不抢焦点。
- 错误文案必须说明原因和下一步，例如“同步失败，网络恢复后可重试”。

## 视觉签名

使用“记忆水纹”作为唯一强记忆点：今日页和训练完成页可以出现低对比水波线或轻微扩散动画，表达记忆被重新唤起。其他区域保持安静，避免整站到处使用渐变、玻璃拟态或浮夸阴影。

水纹动画要求：

- 只在页面进入或训练完成时触发。
- 时长 250-400ms。
- 使用 opacity / transform，不触发布局抖动。
- 遵守 reduced motion，禁用时显示静态水纹。

## 实现前检查清单

- 所有 tappable 元素是否达到 `44x44pt`。
- 底部导航是否不超过 5 项，且图标和文字同时存在。
- 首页是否有唯一主 CTA。
- 首次空状态是否能不登录也开始体验。
- 法务和隐私入口是否在“我的”中可直接到达。
- 图片、语音、同步失败是否有恢复路径。
- 大字号、VoiceOver、低动态、弱网是否有明确验收方法。
- iOS bundle 是否避免关键路径依赖外部 CDN。
