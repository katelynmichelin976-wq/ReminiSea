# 练习模式重设计：普通 + 轻松 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将现有三种练习模式（普通/困难/生存）精简为两种：普通（原生存模式逻辑不动，仅改文案）和轻松（新增认知困难陪伴模式，全牌组巩固，答错不降级）。

**Architecture:** 所有改动在单文件 `yihai_v5.4.html` 内完成。生存模式（`applyCurve(queue)`）的内部逻辑**完全不变**，只把 session_mode 值从 'survival' 重命名为 'normal'（新普通）；删除旧 `applyNormalMode` 函数和 hard 分支；新增 `applyEasyMode` 函数；在 `onNext` 的 learning 重入队列处添加 easy mode 短路；在 SRS write 调用处添加 again→hard 拦截；更新 HTML/i18n/UI 函数。

**Tech Stack:** 单文件 HTML/CSS/JS，Node.js 单元测试（`tests/yihai_v5.0_i18n_test.js`），Playwright 浏览器测试（`tests/_pw_ui_smoke.js`）。

## Task 5: 新增 applyEasyMode 函数

**Files:**
- Modify: `yihai_v5.4.html`（在原 `applyNormalMode` 所在位置插入，即 `buildSessionQueue` 注释行之前）

- [ ] **Step 1: 在 `// ── buildSessionQueue` 注释行之前插入 `applyEasyMode`**

```javascript
// easy mode: full deck pool, familiar-first ordering, no SRS due filter
// familiar = srs_stage 'review' AND lapses_total < 3
// order: [3 warmup familiar] + [3:1 interleave middle] + [2 cooldown familiar]
// wrong answer in session → forced 'hard' (handled at SRS write call site)
function applyEasyMode(cards, stateMap) {
  const SESSION_SIZE = SRS_CONFIG.easy_session_size || 20;
  const today = todayStr();

  // split cards into familiar / unfamiliar pools
  const famPool = [], unfPool = [];
  for (const card of cards) {
    const s = stateMap[card.id];
    if (!s || s.suspended) continue;
    const isFamiliar = s.srs_stage === 'review' && (s.lapses_total || 0) < 3;
    (isFamiliar ? famPool : unfPool).push(card);
  }

  // sort familiar: SRS-due today first (consolidate + due at once), rest random
  for (let i = famPool.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [famPool[i], famPool[j]] = [famPool[j], famPool[i]];
  }
  famPool.sort((a, b) => {
    const sa = stateMap[a.id], sb = stateMap[b.id];
    const aDue = (sa.due_date && sa.due_date <= today) ? 0 : 1;
    const bDue = (sb.due_date && sb.due_date <= today) ? 0 : 1;
    return aDue - bDue;
  });
  // shuffle unfamiliar
  for (let i = unfPool.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [unfPool[i], unfPool[j]] = [unfPool[j], unfPool[i]];
  }

  // if familiar cards are scarce, relax threshold: include non-new non-review cards
  if (famPool.length < SESSION_SIZE - 1) {
    const famIds = new Set(famPool.map(c => c.id));
    for (const card of cards) {
      const s = stateMap[card.id];
      if (!s || s.suspended || famIds.has(card.id)) continue;
      if (s.srs_stage !== 'new' && s.srs_stage !== 'review') famPool.push(card);
    }
  }

  // calculate unfamiliar count: floor((SESSION_SIZE - 5) / 4)
  // e.g. SESSION_SIZE=20 → 3 unfamiliar, 17 familiar
  const unfCount = Math.max(0, Math.min(
    unfPool.length,
    Math.floor((SESSION_SIZE - 5) / 4)
  ));
  const selUnf = unfPool.slice(0, unfCount);
  const selFam = famPool.slice(0, SESSION_SIZE - selUnf.length);

  // build ordered result: [warmup 3] + [3:1 interleave] + [cooldown 2]
  const fam = [...selFam];
  const unf = [...selUnf];
  const warmupCount   = Math.min(3, fam.length);
  const cooldownCount = fam.length >= warmupCount + 2 ? 2 : Math.max(0, fam.length - warmupCount);
  const warmup   = fam.splice(0, warmupCount);
  const cooldown = fam.splice(-cooldownCount);
  const result   = [...warmup];
  let fi = 0, ui = 0;
  while (fi < fam.length || ui < unf.length) {
    for (let k = 0; k < 3 && fi < fam.length; k++, fi++) result.push(fam[fi]);
    if (ui < unf.length) result.push(unf[ui++]);
  }
  result.push(...cooldown);

  return result.map(card => {
    const { options, correct } = buildOptions(card, cards);
    return { ...card, options, correct, _srsState: stateMap[card.id] };
  });
}
```

## 手动验证检查清单

完成所有 Task 后，打开 `http://localhost:8080/yihai_v5.4.html` 手动确认：

1. **设置面板**：打开设置 → 练习模式区域只显示「普通」和「轻松」两项，无「困难」「生存」
2. **普通模式**：选普通 → 开始练习 → 确认卡片按 SRS 到期顺序出现（review due 最早的先出），无额外重排
3. **轻松模式**：
   - 选轻松 → 开始练习 → 出 20 张（或牌组有多少出多少）
   - 开头几张应为已认识的复习卡
   - 答错一张 → 继续到下一张（不重新插入该卡）
   - 打开开发者工具 → Application → IndexedDB → 确认答错的卡 `lapses_total` 未增加
4. **存量用户迁移**：在 localStorage 手动设置 `srs_session_mode = 'hard'`，刷新页面 → 设置面板中「普通」显示 ✓（已迁移）
