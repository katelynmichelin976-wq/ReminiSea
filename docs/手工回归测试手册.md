# 忆海拾光 · 手工回归测试手册

> 用途：把现有自动化回归用例（`tests/_pw_*.js` Playwright 套件）翻译成可照做的手工步骤，供无法跑自动化时人工验证发布质量。
>
> 组织方式：**按测试文件逐节对应**——每节标题即对应的 `_pw_*.js`，方便「某个自动化测试挂了 → 翻到对应节手工复现」。
>
> 单元测试（`tests/run_all.js`，17 套件 706 断言）是纯函数级、无 UI，**无法手工点**，只能 `node tests/run_all.js` 跑，本手册不覆盖。

### 通用操作（下文用【方括号】引用，不再重复）

- **【登录】** 我的 → 点击登录 → 填邮箱+密码 → 勾选「我已阅读并同意…」→ 点登录，等到出现「已登录」状态。
- **【登出】** 账户屏 → 退出登录。
- **【导入牌组】** 设置 → 导入，选 `.yhspack`；或在系统文件管理里打开 `.yhspack` 选「用忆海拾光打开」。
- **【开始练习】** 首页 →「开始练习」。
- **【打开设置】** 首页/我的 → 设置（齿轮）。
- **【切练习模式】** 设置 → 练习模式 → 选「完整 SRS」或「轻松」。
- **【手动同步】** 账户屏 →「同步」按钮。
- **【切高级模式】** 设置 → 模式 →「高级」（底部出现「牌组/统计」Tab）。
- **【刷新】** 刷新浏览器页面（PWA 重新打开 app）。

## 11. `_pw_session_restore` — 会话恢复容错

**前置**：已至少登录过一次（本地有 session backup）。逐个模拟异常场景，每次【刷新】后看账户屏。

| 场景（如何模拟） | 预期 |
|------|------|
| SDK 加载失败（断网/拦截 SDK CDN）后刷新 | UI 不卡在「恢复中」，最终显示登录表单 |
| 无 backup 时刷新 | 显示登录表单 |
| token 已失效时刷新 | 显示登录表单（非一直转圈） |
| backup 数据损坏时刷新 | 不卡死，显示登录表单 |
| 登录请求一直挂起（弱网） | 登录中按钮 disabled；约 15s 超时后按钮恢复可用，显示登录表单 |

> 这是「无感同步」反复打磨的核心容错路径，重点验证**任何异常都不会卡在「恢复中」**。

## 16. `_pw_media_recovery` — 媒体上传失败回滚 + crash 恢复

**前置**：测试账号。需开发者手段模拟「upsert 失败」「crash」（hijack 函数 / 中断同步）。属半自动，纯手工较难，建议优先跑自动化。

| 场景 | 预期 |
|------|------|
| 带图卡同步时 DB 写入失败 | 同步报错；本地图片地址回滚为空、但本地图片 blob 不丢 |

# 第三部分 · 跨设备（需两台设备/两个浏览器档案 A、B）

## 18. `_pw_cross_device` — 跨设备同步综合

**前置**：A、B 登录同一测试账号；导入同一副牌组。

| 步骤 | 预期 |
|------|------|
| 设备 A 练习若干张 → 同步 | A 云端出现 review 卡；A 本地「今日已复习」累加 |
| 设备 B 新登录 | B 自动同步到相同牌组；本地出现 CardState；review 卡不被重置为新卡 |
| 看设备 B 的「今日已复习」 | 为 0（每日进度**不跨设备**，仅本地） |
| 设备 A 编辑单张卡内容 → 同步 | 只有被改的卡内容/时间更新，未改的卡不重传 |

| A 改卡同步、B 拉取 | B 端图片/音频（blob）保留不丢；同步后本地状态仍为「clean」（已同步） |
