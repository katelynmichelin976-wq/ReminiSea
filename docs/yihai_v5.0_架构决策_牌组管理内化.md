# 忆海拾光 v5.0 · 架构决策：牌组管理内化 + 手机制卡

> 决策日期：2026-05-10
> 状态：已确定
> 关联：v4.9.1 已完成部分前置工作（减少逐卡上传、智能同步、白屏修复）

---

## 决策概要

**废弃独立维护 `deck_manager_v1`，将牌组管理功能内化到训练 App 主程序，并在 App 内增加简易制卡入口。**

## 背景

原先的架构将功能拆分到多个 HTML 文件：
- `yihai_v4.x.html` — 训练 App（复习 + 同步）
- `deck_manager_v1.html` — 牌组管理工具（上传 → 合并 → 筛选 → 组织 → 导出）
- `index_v49.html` — 制卡工具（已暂停）

受 Anki 生态启发：创建空牌组 + 导入共享牌组 = 组合功能。这一模式可以替代 `deck_manager` 的核心场景。

## 决策内容

### 牌组管理内化到训练 App

**不再需要独立的管理工具 `deck_manager_v1`，改为训练 App 内直接完成：**

| 功能 | 原先方案 | 新方案 |
|------|---------|--------|
| 导入 `.yhspack` | 训练 App 已有 | 保留，不变 |
| 创建空牌组 | ❌ | App 内加按钮 |
| 从服务器拉取牌组 | 云端 Tab 浏览 | 强化为"添加到我的牌组"（多牌组共存） |
| 多牌组组合 | `deck_manager_v1` 筛选合并 | App 内直接导入/拉取多个牌组共存 |
| 编辑/删除卡片 | `deck_manager_v1` | 陪护者通过**浏览牌组功能**删除 + 管理端 `yihai_admin_v1` 批量改 |

### 手机制卡替代 `index_v49`

制卡工具不再需要 PC 端独立页面：

| 制卡方式 | 手机可行性 |
|---------|-----------|
| 拍照 | ✅ 比 PC 方便 |
| 录音 | ✅ 比 PC 方便 |
| 文字 | ✅ 手机输入足够 |
| 批量导入 | 仍通过 `.yhspack` 文件走桌面流程 |

### 管理端保持独立

`yihai_admin_v1` 继续保留，承担陪护者/医生端的编辑、删除、批量修改职责。

## 完整闭环

```
制卡（App 内拍照/录音，或导入 .yhspack）
  → 保存到"我的牌组"
  → 同步到服务器
  → 多设备复习
  → 陪护者通过浏览牌组功能 或 管理端(yihai_admin_v1) 编辑/删除
```
