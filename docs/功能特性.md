## 一、产品形态

- **单文件 PWA** — 全部代码内联在 `index.html`（CSS/JS 不外置），可添加到主屏幕，行为接近原生 App。
- **离线优先** — 卡片导入后全程无需联网即可学习；登录仅用于多设备同步。
- **本地存储** — localStorage（牌组索引/设置/SRS 配置/每日进度）+ IndexedDB（卡片状态、答题流水、媒体 blob、Easy 状态、本地日志）。
- **云端可选** — Supabase 账号登录后跨设备同步学习记录与牌组。
- **Service Worker** — 当前暂未启用（GitHub Pages App-Bound Domain 限制，待绑独立域名）。

## 二、学习与练习

### 2.1 SRS 间隔重复引擎
- 自研 SM-2 变体，三阶段状态机 `learning → review ← relearning`（`processAnswer`）。
- 四级评分对齐 Anki 命名（again/hard/good/easy），参数不加后缀。
- 关键保护机制：
  - `daily_remove_lapses`（默认 3）：当天连续失败 N 次移出队列。
  - `auto_suspend_lapses`（默认 8）：累计失败 N 次自动挂起。
  - `learn_ahead_limit`（默认 1200s）：防跳过 learning 步骤。
- 每日额度：`new_cards_per_day`（5）、`maximum_reviews_per_day`（50）。
- 完整状态机与公式见 `srs_design_v6.9.md`。

### 2.2 练习模式（`SRS_CONFIG.session_mode`）
- **normal（完整 SRS）** — 全量到期积压，按 `due_ts` 升序出牌（Anki 默认顺序，无重排）。
- **easy（陪伴模式，v5.11 重设计）** —
  - 独立 EasyState 数据层（IDB `easyCardStates` + 服务器 `easy_card_states` trigger 跨设备同步）。
  - 三级分类 confident / learning / unseen（`history === [1,1,1]` 才算 confident）。
  - Session 结构 `[warmup CCC] + (L CCC)×k + tail r×C`，按 T 自适应；预设规模 chip 15/19/23（`easy_session_size`）。
  - 答错可重试开关 `easy_retry_on_wrong`（默认 true）。
  - 首错以 `_pendingAttempt` 为唯一真相源记 0；不写 `sync_card_states`。

### 2.3 答题交互
- 选择题：点对变绿进入下一张，点错淡出错项可再选（easy 模式可关）。
- 自动推进：`practice_advance_sec`（默认 6s，0 = 手动）。
- 时长守护（仅练习模式）：`idle_threshold_sec` 空闲检测、`warn_duration_sec`/`warn_repeat_sec` 长时间提醒（warn/limit 两档）。
- 完成屏：单局统计汇总。

### 2.4 浏览模式
- `screen-browse` 翻看牌组卡片，与练习（`screen-quiz`）分离。

## 三、牌组管理

### 3.1 三段式牌组页（`screen-decks`）
- **本地** / **云端**（`screen-cloud-decks`）/ **精选** 三段。
- 牌组详情 `screen-deck-detail`、新建卡片 `screen-create-card`。
- 本地牌组更新提示：session 就绪后比对云端 `decks.updated_at` 打黄点（`refreshDeckUpdateBadges`）。

### 3.2 牌组类型（`deck_type`）
| 类型 | 来源 | 可见性 |
|------|------|--------|
| `preset` | 管理员预置 | 全员可读 |
| `shared` | 用户发布（待设计） | 全员可读 |
| `personal` | 用户本地创建并上传 | 仅 owner |

### 3.3 卡片类型
- 图片卡、语音卡、文字卡（图片/音频/文字三类内容）。
- 多语言卡片支持 `deckLang` 标记，用于 TTS 语言识别。

### 3.4 导入 / 导出
- `.yhspack`（ZIP）格式导入：`deck.json` + `media/`。
- 个人牌组可上传至云端并跨设备同步（结构/卡片正文/媒体三阶段，见 architecture.md）。

## 四、语音辅助

- 全屏管理页 `screen-voice-assist`，录制/编辑语音槽（`VOICE_SLOTS`）。
- **情感语音槽**（可录音，emotion）：答错提示、答对提示、连对鼓励、整局结束。
- **功能语音槽**（functional）：题干朗读、选项提示。
- TTS：浏览器语音合成；`warmupSpeech()` 须在用户手势内调用（iOS 解锁 TTS + Audio）。
- `sessionId` 机制：每次启动/回首页递增，打断跨页异步 TTS 链。

## 五、云同步与账户

### 5.1 账户（`screen-account`）
- 邮箱 + 密码登录/注册（账号不存在自动创建），密码重置 `screen-reset-password`。
- Supabase SDK 自动持久化 session；启动时 `restoreSession()` 恢复。
- 状态门控：`_syncEnabled`（在线）、`_sessionRestoring`（恢复中）。
- 设备登记 `device_registry`（first_seen / last_seen）。

### 5.2 同步内容
- **逐卡上传** — 仅 `sync_trials`（答题流水）；`sync_card_states` 由 DB trigger 自动维护。
- **统一入口** — 所有同步走 `runSync(options)`（`modal` 控弹窗，`decks` 控牌组同步）。
- **配置同步** — `sync_config`（语音参数等）跨设备（`cloudPushConfig`/`cloudPullConfig`）。
- **同意状态同步** — 隐私协议同意跨设备 + 版本升级弹窗（`showConsentUpgradeDialog`）。
- **仅本地（不跨设备）** — `daily_progress`（reviewed_today/daily_new_today）、Easy 模式 `last_warmup`。
- 看门狗：`runSync` 30s watchdog 防卡死。

### 5.3 跨设备一致性
- CardState / TrialLog 通过云端同步；review 不被旧设备覆写；增量同步 + 暂停续传。

## 七、个性化与多语言

### 7.1 界面模式（`appMode`）
- **standard** — Tab Bar 显示「首页 / 练习 / 我的」3 项。
- **advanced** — 显示「首页 / 牌组 / 练习 / 统计 / 我的」5 项（`.advanced-only` 控制）。

### 7.2 多语言 i18n（`screen-lang`）
- 5 种语言：简体中文（zh-CN）、英文（en）、繁体中文（zh-Hant）、西班牙文（es）、日文（ja）。
- PP/ToS 协议链接随 locale 路由（zh-CN/en/zh-Hant，es/ja 回退）。

### 7.3 主题（`screen-theme`）
- 5 套配色：默认（default）、琥珀（amber）、朱砂（cinnabar）、暗色（dark）、翡翠（jade）。

### 7.4 SRS 参数自定义
- 全部 `SRS_CONFIG` 参数支持用户编辑并持久化（localStorage override，启动时恢复）。

## 八、隐私与反馈

- **隐私同意** — 登录/注册表单强制勾选，落地 localStorage；协议版本升级时弹窗重新确认。
- **意见反馈**（`openFeedbackSheet`/`submitFeedback`）— 写入 `feedback` 表（anon INSERT）。
- **本地日志** — IDB 日志 ring buffer；JS 异常自动上报（window.error + unhandledrejection → `appEvents`，session 去重，`purgeOldLogs` 定期清理）。

## 九、健壮性设计

- **无 confirm()** — iOS PWA 会阻塞，改用 `showConfirmDialog()` 自定义弹窗。
- **SRS 写竞态保护** — `_lastSrsWrite` promise chain；回首页/开统计须先 `await`。
- **IDB 写入容错** — 写入函数抛错不掐断答题流。
- **媒体确认状态** — `card.media[slot].confirmed` 仅本地持久化，crash 恢复时自动补 DB 写（不重传 Storage）。
- **媒体批量提交** — `upsertCardsMediaBatch` 替代逐 slot upsert，避免 N×M 次往返。
- **诊断面板** — `yh_diag`（CDN 加载）开发期诊断工具。
