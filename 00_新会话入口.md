## 六、关键架构决策

1. **单文件原则** — 所有工具都是独立 HTML 文件
2. **simpleHash 一致性** — 制卡工具、管理工具、训练端共用相同实现
3. **卡片不做自动去重** — 用户手动整理
4. **合并策略默认「保留两者」** — 最安全
5. **训练端同步静默非阻塞** — 所有 Supabase 调用 try/catch，失败不影响训练
6. **增量同步** — 基于 cards_pool.updated_at + 本地 _imgUrl/_audUrl 对比，只拉变更
7. **会话持久化** — Supabase SDK 自动管理 session，重启恢复
